Multi-Source Research Assistant

Multi-Source Research Assistant

An AI-powered document assistant that enables users to upload PDFs and query them using natural language.

The system uses Retrieval-Augmented Generation (RAG) to retrieve relevant document chunks and generate accurate, context-aware answers with source references. Uploaded documents are automatically chunked, embedded, and indexed in a vector database to support semantic search.

When a query is asked, the system retrieves the most relevant sections and combines them with a language model to produce reliable responses.

The application also displays retrieval routes, source citations, and response timings, improving transparency and helping users understand how answers are generated.

  • Tech Stack: Python, FastAPI, LangChain, ChromaDB, OpenAI API, Vector Search
  • Focus: AI / Machine Learning / Backend Systems / Document Intelligence

Hospital Readmission Prediction

Developed a machine learning pipeline to predict 30-day hospital readmissions for diabetic patients using the UCI Diabetes dataset.

The project focuses on handling imbalanced healthcare data, feature engineering, and model evaluation to improve predictive performance.

Multiple sampling strategies were implemented, including random undersampling, stratified undersampling, and SMOTENC oversampling, alongside Gradient Boosted Trees for classification.

The models were evaluated using AUC, precision, recall, and F1-score, with experiments conducted in Apache Spark and PySpark to support scalable data processing.

  • Tech Stack: Python, PySpark, Scikit-learn, Apache Spark, Imbalanced-learn
  • Focus: Data Science / Machine Learning / Healthcare Analytics

Driver Fatigue Detection System

Developed a machine learning system for detecting driver fatigue using physiological signals, combining EEG and eye-movement features from the SEED-VIG dataset.

The goal of the project was to identify patterns associated with fatigue and build predictive models that could support real-time driver monitoring systems for improved road safety.

The project involved extensive data preprocessing, feature exploration, and statistical analysis of EEG frequency bands and eye-movement indicators such as blink patterns and PERCLOS-based fatigue labels.

Feature transformation and dimensionality reduction techniques were applied to improve signal quality and model performance.

Multiple machine learning models were trained and evaluated to classify fatigue states, with experiments focusing on feature engineering, model comparison, and performance evaluation using standard metrics.

The work demonstrates how physiological signals and machine learning can be combined to develop AI-driven safety systems in intelligent transportation and human-state monitoring applications.

  • Tech Stack: Python, Scikit-learn, Pandas, NumPy, Signal Processing
  • Focus: Artificial Intelligence / Machine Learning / Biomedical Data Analysis / Human State Detection

Priority-Based Appointment System (NHS)

Developed a data-driven appointment prioritization system designed to improve patient scheduling efficiency within healthcare settings.

The project focused on using data analysis and algorithmic prioritization to ensure patients with higher medical urgency are scheduled earlier while maintaining overall system efficiency.

The system analyzes patient data such as symptoms, urgency level, and waiting time, and assigns priority scores to optimize appointment allocation.

By introducing a structured prioritization model, the system helps reduce waiting times for critical cases and improves fairness in appointment scheduling.

The project demonstrates how data science and algorithmic decision-making can support healthcare operations, improving resource utilization and patient service delivery.

  • Tech Stack: Python, Data Analysis, Scheduling Algorithms
  • Focus: Data Science / Healthcare Analytics / Decision Support Systems
